Exploring the GATK and Samtools datasets

SNPs were called from 266 salmonella isolates using two methods, GATK and Samtools. The input for each SNP calling pipeline is fasta files. The output from each pipeline is a list of the isolates and the number of SNPs called from each.

The data used for exploring the results of these two methods combines the genome ID, the number of SNPs called for each method, and serotype code information.

An initial scatterplot comparing GATK and Samtools SNP calling results shows that most of the data clusters in a roughly linear correlation. There are several outliers to be examined more closely. At this resolution, there doesnt appear to be clustering on serotype code.

We can see that the two datasets have roughly the same shape. Additionally, they have almost the same number of outliers (GATK: n=42, Samtools: n=41).

Density plots dont really give us additional information, but some people prefer them over boxplots.

Eliminating the visually obvious outliers gives us a better picture of the overall correlation. There still appears to be a linear correlation, although the Samtools method has generated roughly three times as many SNPs.

Dividing by 3 the number of SNPs called by Samtools, we get a plot with the same shape.

Looking at the outliers

There are 8 visually-obvious outliers that we removed to generate the last graph.

One of these is the genome that the Samtools method couldnt complete (Sal_JR3_62-z36_RKS2983). The de-duped bam file appears fine, and runs fine through the GATK pipeline (33,424 SNPs called). bcftools mpileup seems to generate a corrupt bcf file, and when bcftools call is run, it gives the error: No BGZF EOF marker; file may be truncated.

Interestingly, the three genomes with serotype code JEG called roughly the same number of SNPs for each method. Im not sure what to think of this, since the Samtools method called 3x as many SNPs for most of the isolates.

A scatter plot color-coding size shows that the outliers are all files in the top 25% of the data. However, there are also many files in the top 25% that lie along the line. All the files in the middle 50% fall along the line, and most of the smallest 25% of files do as well. There are a few exceptions among the small files that we could examine further

## 
##  small medium  large 
##     68    134     68

It is safe to say that, for this dataset, GATK and Samtools capture a proportionately appropriate number of SNPs for most samples. The exceptions are verry large files and some very small files. Things to look into:

  1. Why Samtools captures 3x more SNPs on average than GATK.
  2. Whats up with the very large files?
